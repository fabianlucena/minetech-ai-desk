- El cliente se comunica siempre vía WhatsApp
- Los técnicos (que están de guardia) interactúa a través de una App mobile a desarrollar en el presente proyecto
- Toda la comunicación debe quedar registrada
- Identificar al cliente con un código de cliente
- En cada comunicación registrar Nombre, número de teléfono y email del que llama
- El cliente debe identificarse con un token
- En un backoffice poner una interface para:
  - roles:
    - Administrador: puede acceder a un dashboard, esquemas de guardias, gestión de clientes, gestión de guardias
    - Supervisor: seguimiento de tickets
  - dar de alta al cliente y darle un token de por ejemplo 3 letras y 5 números (u otra propuesta más sencilla de cara al cleinte) para que se identifique: por ejmplo: CSJ-56812, MBA-00265, JSM-99801, etc 
  - Guardar un esquema de guardias
  - Colocar un dashboard de tickets por cliente
  - métricas: tiempo de respuesta y tiempo de resolución
- El cliente debe poder listar sus tickets abiertos y cerrados
- El técnico también debe poder ver el historial de consultas del cliente
- Debe haber un técnico de guardia principal y uno de reserva en caso que el principal no se pueda cominucar.
- Se realizan menos de 10 consultas diarias
- Cada comunicación se crea un ticket, si es para continuar un ticket anterior se vinculan
